## 🏛️ 第一章：项目的“三层汉堡”架构

现代软件开发绝对不能把所有逻辑揉成一坨，必须各司其职。我们的 MiniNoteHub 采用了超经典的**三层架构**：

| 层级名称 | 代表文件 | 职责说明 (人话) | 必须遵守的铁律 |
| :--- | :--- | :--- | :--- |
| **Domain (领域层)** | `Note.hpp` | 就像一张**真实存在的便签纸**。它只负责存自己的数据（标题、正文、时间）。 | 绝对不准碰磁盘文件，绝不知道“仓库”的存在！ |
| **Data (数据层)** | `repository.hpp` | 就像一个**带锁的抽屉**。专管把多张便签纸收集到一起（增删查改）。 | 使用 `std::vector` 高效管理卡片。 |
| **Persistence (持久层)** | `Storage.hpp` | 就像**档案科员工**。下班时把抽屉里的卡片序列化到硬盘，上班时再读回来。 | 负责翻译工作：面条变拉面（字符串 <-> 对象）。 |
| **App (应用层)** | `main.cpp`/`command.hpp` | 就像**驾驶室和方向盘**。负责跟人类交互，解析人敲的命令。 | 控制死循环，连接上面三大模块。 |

## 💡 第二章：核心语法拆解（你踩过的坑都在这）

### 1. `enum class` 是个什么神仙？
**问题**：传统 C++ 里的 `enum`（枚举）是个马大哈，里面的名字会污染整个环境，而且很容易被当成普通的 `int` 乱加乱减。
**解法**：`enum class CommandType` 是强类型枚举！
*   **它的好处**：它像是一个独立的房间。你要用 `ADD` 必须写出它的全名 `CommandType::ADD`。
*   **为什么一定要用？** 它直接杜绝了你把“未知命令”不小心和一个随便的数字相加引发的血案。它让编译器的红色波浪线成为保护你的防弹衣！

### 2. `stoll` 的魔术 (String TO Long Long)
我们在 `Storage` 读文件（Load）时遇到了一个难题：从文件里读出来的时间戳是一串字符 `"1713182800"`。但是卡片对象要求的是一个时间数字。
*   **出场武器**：`std::stoll(字符串)`。
*   **作用**：它像个破壁机一样，瞬间把文本形式的 `"170000"` 榨汁还原成了可以做时间计算的 `long long` 超级大整数。

### 3. 为什么疯狂使用 `size_t`？
永远记住这个结论：**只要是表示“第几个”、“数组有多大”，必须用 `size_t`。**
*   这玩意是无符号的，永远不会是负数。
*   它能保证你找卡片（`Notes[index]`）时，永远不会因为误传了一个负数直接导致内存越界、程序崩溃。

## ⛓️ 第三章：IO 文件流 — 烤串与切西瓜

文件读写的本质：
*   **写入 (Save) 就是烤串**：你有一个对象（卡片），你把它上面的肉（标题）、大葱（正文）、青椒（时间）用竹签（分隔符 `|`）串起来：`out << title << "|" << content << "\n";`
*   **读取 (Load) 就是切西瓜**：你拿到一整串文本，用 `std::stringstream` 把它装起来，然后用 `std::getline(ss, 目标变量, '|')` 就像菜刀一样，切到 `|` 就自动停下。

## 🚀 第四章：控制台死循环主函数 `main` 的美学

你的 `main.cpp` 有一套属于自己的“心跳节奏”：
1. **开机加载**：`storage.load(repo)`
2. **死循环开始**：`while (true)`
3. **倾听命令**：`std::getline(std::cin, line)`
4. **命令翻译官**：交给 `CommandParser`，拿到 `enum class`
5. **分发执行**：用到超经典的 `switch (cmd.type)` 结构。
6. **关机存档**：打出 `quit` -> `storage.save(repo)` -> `break` 结束心跳。
